## Why This Model

Most forecasting fails because it relies on rep intuition alone. This model combines:

- **Historical conversion patterns** by stage and segment
- **Deal-specific signals** like engagement and timing
- **Manager judgment** with structured overrides

The result is a forecast that's both data-driven and human-informed.

### How It Works

1. **Import Pipeline** - Paste your current pipeline from CRM
2. **Apply Weights** - The model applies probability weights based on stage and signals
3. **Add Judgment** - Managers can override individual deals with context
4. **Run Scenarios** - See best case, worst case, and expected outcomes
5. **Track Accuracy** - Compare forecasts to actuals to improve over time

### Key Features

**Weighted Probability**
Each deal is weighted based on:
- Stage in pipeline
- Days in stage
- Recent activity
- Deal size (larger deals get conservative weighting)

**Scenario Modeling**
Run three scenarios simultaneously:
- Best case (all commit deals close)
- Expected (weighted probability)
- Worst case (only high-confidence deals)

**Accuracy Tracking**
Track forecast accuracy over time:
- By rep and manager
- By deal size and segment
- By forecast category
